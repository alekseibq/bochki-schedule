# Agent workflows

Эти правила загружай, когда пользователь просит выполнить стандартный agent-workflow вроде `ship-pr` / `ship-pr-no-fix` или другую повторяемую цепочку действий, которую нужно выполнять по зафиксированному сценарию.

## Команды

- `ship-pr`:
  - стандартный workflow `commit -> push -> PR -> wait for CI`;
  - по умолчанию включает `auto-fix simple CI issues`;
  - перед `git add` агент должен проверить и явно определить состав файлов, которые пойдут в commit;
  - источником истины для результата считается GitHub Actions.
- `ship-pr-no-fix`:
  - выполняет тот же workflow;
  - если CI падает, агент не делает автофиксы и сразу возвращает причину падения.

## Обязательное поведение

- Для механической части агент должен использовать repo-local scripts:
  - `scripts/agent/ship-pr.sh`
  - `scripts/agent/wait-ci.sh`
- Если работа начинается на `main`, агент должен создать отдельную ветку и не коммитить напрямую в `main`.
- Commit message должен быть в формате Conventional Commits.
- `PR title` по умолчанию совпадает с commit message.
- `base branch` по умолчанию `main`.
- В commit нельзя включать временные untracked-файлы без явного запроса пользователя.

## Auto-fix policy для `ship-pr`

- Разрешены только простые исправления:
  - `prettier` / format fixes;
  - небольшие lint fixes;
  - локальные test/type fixes без расширения исходной задачи;
  - мелкие правки, напрямую устраняющие конкретный failing CI step.
- Не разрешены без нового явного запроса:
  - новые runtime/development dependencies;
  - архитектурные или широкие рефакторинги;
  - изменения публичного поведения, не требуемые исходной задачей.
- После каждого автофикса агент повторяет цикл `commit -> push -> wait for CI` в той же ветке и PR.

## Stop conditions

- `ship-pr` должен остановиться и вернуть причину, если:
  - падение CI не относится к allowed auto-fix scope;
  - для исправления нужен продуктовый выбор;
  - CI продолжает падать по сути на той же причине после 1-2 локальных итераций.
- `ship-pr-no-fix` должен остановиться сразу после первого красного CI без попыток исправления.
