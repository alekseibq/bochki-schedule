# Contributing rules

Эти правила загружай, когда задача касается внесения изменений в код, структуры коммитов, Pull Request, веток, review-процесса и правил внесения правок в репозиторий.

- Коммиты оформляй в формате Conventional Commits.
- В `main` не вноси изменения напрямую. Любые правки должны проходить через отдельную ветку и Pull Request.
- Перед внесением правок проверь текущую ветку и состояние рабочей копии, чтобы не перетереть чужие изменения.
- Новые runtime/development dependencies добавляй только после явного согласования.
- Если процесс требует отдельного правила, добавляй его сюда или в более специфичный файл, а не в `README.md`.
