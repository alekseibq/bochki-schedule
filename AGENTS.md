# AGENTS.md

Этот файл является основным источником истины для ИИ-агентов в этом репозитории.
`README.md` предназначен для людей и не является источником истины для агентов.

Правила ниже разделены по областям применения и вынесены в отдельные файлы, чтобы не тащить лишний контекст в каждый запрос к LLM.
Перед выполнением задачи загружай только те подразделы, которые относятся к текущей работе.
Если задача затрагивает несколько областей, загружай все соответствующие файлы.
Если между файлами есть противоречие, более специфичный файл имеет приоритет над более общим.

## Терминология домена

- Во внутренних сущностях, типах и persisted-данных используй английские имена: `Trainer`, `Participant`, `ProcedureType`, `Seminar`, `ProcedureSession`.
- В пользовательском интерфейсе используй русские подписи.
- Для этого проекта `Trainer` в модели соответствует пользовательскому термину `Сопровождающий`.
- Термин `Тренер` считается допустимым синонимом в обсуждении задач, issue и документации, но не является основным UI-лейблом.

## Правила выполнения зависимых операций

- Если одна команда изменяет состояние сущности, а следующая команда проверяет результат этого изменения для той же сущности, выполняй их только последовательно.
- Не запускай параллельно `write` и `read` для одного и того же GitHub-объекта, git-состояния, файла или другого общего ресурса.
- Для сценариев вида `изменить и сразу проверить` сначала дождись успешного завершения операции изменения, затем отдельно выполни верификацию.

## Правила по вкладу в репозиторий

- [agent-rules/CONTRIBUTING.md](./agent-rules/CONTRIBUTING.md) - загружать, когда задача касается внесения изменений в код, структуры коммитов, Pull Request, веток, review-процесса и правил внесения правок в репозиторий.
- [agent-rules/RELEASE-WORKFLOW.md](./agent-rules/RELEASE-WORKFLOW.md) - загружать, когда задача касается CI/CD, release workflow, ручного QA desktop-приложения, GitHub Actions artifacts/Releases и доставки проверенных артефактов заказчику.
- [agent-rules/WORKFLOWS.md](./agent-rules/WORKFLOWS.md) - загружать, когда пользователь просит выполнить стандартный agent-workflow вроде `ship-pr` / `ship-pr-no-fix` или другую повторяемую цепочку действий, которую нужно выполнять по зафиксированному сценарию.

## Правила по разрешениям и рабочему циклу агента

- [agent-rules/AGENT-LOOP-PREMISSIONS.md](./agent-rules/AGENT-LOOP-PREMISSIONS.md) - загружать, когда задача касается выполнения команд в CLI/shell, разрешений на `git commit`, `git push`, `gh run list`, `gh run watch`, а также других действий, требующих отдельной политики подтверждения.
